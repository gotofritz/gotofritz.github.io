Javscript Performance Mythbuster summary

Summary of the Javascript Performance Mythbusters slideshow, conveniently in one page. Hope this doesn’t breach any copyright. A great presentation @ Velocity 2012 conference on checking common assumptions. With work by Lindsey Simon, Kyle Simpson, Chris Joel, Nicholas C. Zakas

Busted myths

  • String concatenation is not slower than Array.join
  • Creating a String object is not faster than assigning a string value to a variable
  • Local storage is not much slower than cookies
  • Eval doesn’t necessarily turn off optimization / Yes it does (either way, I don’t go anywhere near it if I can help it)
  • Sorry Dan Webb, but hashbang routing is still needed because history.pushState is not there yet
  • A neat trick to improve jQuery looping - and the proof it works
  • querySelectorAll() is NOT faster than getElementsByTagName() - and here’s why
  • a good idea to cache a nodeList to an array
  • script defer is buggy on IE. A good strategy is to concat the files, and then chunk in 100k chunks based on how often they change, perhaps with different caching headers and with lay loading
  • some simplified ES5 shims can be faster than native implementations: object creation, forEach, bind
  • RegExp are NOT slower than String.indexOf
