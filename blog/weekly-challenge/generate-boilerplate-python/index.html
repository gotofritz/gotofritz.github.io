About the challenge

Haven’t been doing one of these for a while, due to a new addition to the family. I need a very easy one to get me back into it.

Setting up a weekly challenge is a repetitive task, so I should write a script for it. I picked Python as the scripting language.

Some artefacts available on GitHub.

Weekly Challenge Generator

A single feature should be enough to describe this.

Setting up script

The easiest of first steps. In mock TDD style, I write only enough code to pass the “tests” - except that there are no test scripts yet.

Saved the script as weekly, made it executable, and created a symbolic link to it from /usr/local/bin

Then tested by going into the directory weekly-challenges and typing weekly - it works.

Copying boilerplate

I created a folder called boilerplate, with the three required file. The task is simply to copy it over and rename it. A little bit of messing about with paths, but in essence a trivial task. Passing “yyy” as an argument was added retroactively to keep it consistent with later scenarios.

Naming the new directory

I used the argparse module to parse the command line arguments, and a couple of regular expressions to remove punctuation from the argument.

This works for all challenge names passed.

Numbering challenges

Now the issue is reading existing folder names, detecting the number in it, increasing it, and using it for the new folder name.
